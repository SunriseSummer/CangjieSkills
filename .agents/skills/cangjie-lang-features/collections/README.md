# 仓颉集合数据类型

## 1. 选型速查

| 类型 | 长度/容量 | 元素/键约束 | 典型复杂度 | 适用场景 |
|------|-----------|-------------|------------|----------|
| `Array<T>` | 长度固定 | 无额外约束 | 下标访问 `O(1)` | 已知长度、强调顺序访问、需要切片 |
| `ArrayList<T>` | 自动扩容 | 无额外约束 | 末尾追加均摊 `O(1)` | 动态收集元素、频繁尾部追加 |
| `HashMap<K, V>` | 动态 | `K <: Hashable & Equatable<K>` | 查找/插入均摊 `O(1)` | 按键查值、去重索引、计数 |
| `HashSet<T>` | 动态 | `T <: Hashable & Equatable<T>` | 查找/插入均摊 `O(1)` | 去重、成员测试、集合运算 |

## 2. 高频语义差异

- **`Array` 是固定长度顺序容器**：适合“长度确定 + 下标访问”的场景；切片、复制和原地修改语义最重要。
- **`ArrayList` 是动态顺序容器**：适合构建结果集、批量追加、需要插入删除但仍保持顺序的场景。
- **`HashMap` 保存键值对**：键必须可哈希且可判等；遍历顺序不应作为业务语义依赖。
- **`HashSet` 只关心元素存在性**：本质上是“只存 key 的 HashMap”，适合去重和快速包含判断。

## 3. 常见选择建议

1. **已知长度且要频繁按索引读写** → 优先 `Array`
2. **长度未知，需要不断 `add()`** → 优先 `ArrayList`
3. **需要“键 → 值”映射** → 使用 `HashMap`
4. **只需要去重 / contains** → 使用 `HashSet`
5. **既要顺序又要去重** → 常见做法是 `ArrayList + HashSet` 组合维护
